Great Indian Grocery – E-Commerce Platform Implementation & QA Case Study

Overview

This repository documents the end-to-end product implementation, UX restructuring, backend configuration, and quality validation of the Great Indian Grocery WooCommerce-based e-commerce platform prepared for production launch.

This project focuses on product structure, user experience optimization, checkout workflow enhancements, inventory configuration, and structured release validation rather than modification of WooCommerce core source code.

Platform Stack

- WordPress (PHP)
- WooCommerce
- Elementor
- MySQL
- Hostinger hosting environment

Scope of Work

- Homepage UX restructuring for improved product discoverability
- Departmental taxonomy and category optimization
- Configuration of 2,000+ SKUs with structured metadata
- Checkout workflow enhancement including curbside pickup metadata collection
- Promotional campaign timer configuration
- Backend inventory and category logic configuration
- Functional validation of cart logic and checkout behavior prior to production release

Role & Responsibilities

- Led product structure redesign and navigation flow improvements
- Defined and implemented custom checkout metadata workflow
- Configured backend product categories and inventory workflows
- Conducted functional and regression validation cycles
- Directed pre-release content and workflow verification

Quality & Validation Approach

- Functional testing across product, cart, and checkout flows
- Regression validation after backend configuration updates
- Edge-case validation for metadata capture and session continuity
- Release-readiness checklist prior to production deployment

Repository Structure

/docs – Case study documentation  
/screenshots – UI and backend screenshots (sanitized)  
/qa-validation – Structured test plan and regression checklist  
/release-notes – Iterative release tracking  

Status

In progress – Production launch preparation.
